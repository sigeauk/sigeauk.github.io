TIDE

Threat Informed Detection Engineering

A standalone, containerized platform for managing the full Detection Engineering lifecycle—from threat intelligence to validated SIEM rules.

Architecture Overview

TIDE utilizes a robust, three-tier architecture designed for air-gapped or enterprise environments:

  • Nginx: Handles SSL termination and reverse proxying.
  • FastAPI: Powers the core REST API and HTMX dynamic frontend.
  • DuckDB: Embedded analytics database for high-performance rule caching.

Deployment Guide

Prerequisites: Docker & Docker Compose v2+ installed. API access to your Elastic Security (Kibana) instance.

1. Environment Variables

Configure your connection to Elastic SIEM and optional services like Keycloak or OpenCTI.

.env 
TIDE_VERSION=2.4.0
APP_URL=https://tide.yourdomain.com

# Elastic Security Configuration
ELASTIC_URL="http://kibana:5601"
ELASTICSEARCH_URL="http://elasticsearch:9200"
ELASTIC_API_KEY="your-secret-api-key"

# Optional Integrations
AUTH_DISABLED=false
OPENCTI_URL="http://opencti:8080"
GITLAB_URL="http://gitlab:8929"

2. Docker Orchestration

TIDE runs as a multi-container stack. The app container persists its state in the ./data volume.

docker-compose.yml 
version: '3.8'
services:
  tide-app:
    image: 047741/tide-core:${TIDE_VERSION}
    container_name: tide-app
    env_file: .env
    volumes:
      - ./data:/app/data
    expose:
      - "8000"
    networks:
      - tide-network
    restart: unless-stopped

  tide-nginx:
    image: nginx:alpine
    container_name: tide-nginx
    ports:
      - "443:443"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/conf.d/default.conf:ro
      - ./certs/server.crt:/etc/nginx/certs/server.crt:ro
    networks:
      - tide-network
    depends_on:
      tide-app:
        condition: service_healthy

networks:
  tide-network:
    driver: bridge

3. Launch

Start the production stack with a single command:

docker compose up -d

Data Persistence

TIDE is designed for stability. The following files are maintained in your persistent volume to survive container updates:

  • tide.duckdb: The central rule cache and coverage metrics.
  • checkedRule.json: Manual analyst validation records.
  • triggers/: Directory for background synchronization state.
